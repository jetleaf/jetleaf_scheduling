MAX_CONCURRENCY_PROPERTY_NAME constant

String const MAX_CONCURRENCY_PROPERTY_NAME

Environment property name for configuring maximum concurrent task executions.

This property controls the maximum number of scheduled tasks that can execute simultaneously. This helps prevent resource exhaustion when many scheduled tasks are configured.

Property Format: jetleaf.scheduler.maxConcurrency Expected Value: Positive integer Default: Platform-dependent (typically based on available processors)

Example configuration:

jetleaf.scheduler.maxConcurrency=10

Usage in code:

final maxConcurrency = environment.getPropertyAs<int>(
  SchedulingAnnotationPodProcessor.MAX_CONCURRENCY_PROPERTY_NAME, 
  Class<int>(null, PackageNames.DART)
);
